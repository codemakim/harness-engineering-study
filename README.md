## 개편 원칙

새 문서는 단순 요약이 아니라 기초부터 실력을 만드는 강의 자료여야 한다.

각 장은 가능하면 다음 흐름을 따른다.

```text
1. 먼저 직관
2. 정확한 개념
3. 웹 개발자 비유
4. 작은 예시
5. 왜 중요한가
6. Codex에서는 어떻게 보이는가
7. 흔한 오해
8. 요약
9. 생각해볼 질문
```
